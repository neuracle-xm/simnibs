# 任务进度持久化与断点恢复设计方案

## 一、需求

RabbitMQ 消息处理中，如果程序在任务执行过程中崩溃或断连，消息不会被 ack，RabbitMQ 会自动重发。需要进度持久化实现断点恢复，避免重复执行已完成步骤。

**注意**：

- 进度枚举（ModelProgress、ForwardProgress、InverseProgress）已在 `neuracle/rabbitmq/progress.py` 中实现
- 进度文件读写函数（load_progress、save_progress）已在 `neuracle/rabbitmq/progress.py` 中实现
- 消息的 `redelivered` 属性会传递给任务处理函数
- **但目前实际恢复逻辑尚未激活**（redelivered 参数当前未被使用来跳过已完成步骤）

## 二、简化方案

### 2.1 进度文件位置

在 Model 任务输出目录中创建进度文件：

```
data/
└── m2m_{subjectid}/
    └── .progress.txt                    # Model 任务进度
```

### 2.2 进度文件格式

每个进度文件只记录当前进度值：

```
35
```

文件内容就是当前进度枚举的整数值。

### 2.3 进度枚举定义

```python
from enum import IntEnum

class ModelProgress(IntEnum):
    """Model 任务进度枚举"""
    START = 0          # 任务开始
    PREPARE_T1_DONE = 10   # prepare_t1 完成
    PREPARE_T2_DONE = 20   # prepare_t2 完成
    DENOISE_DONE = 35      # denoise 完成
    INIT_ATLAS_DONE = 50   # init_atlas 完成
    SEGMENTATION_DONE = 70 # run_segmentation 完成
    SURFACES_DONE = 85     # create_surfaces 完成
    COMPLETED = 100        # 全部完成
```

## 三、实现逻辑

### 3.1 新任务开始

1. 检查消息的 `redelivered` 属性
2. 如果 `redelivered == False`（新消息），从头开始执行
3. 如果 `redelivered == True`（重发消息）：
   - 读取进度文件获取当前进度
   - 从当前进度继续执行

### 3.2 步骤完成

1. 更新进度文件内容为当前进度枚举值
2. 继续执行下一步骤

### 3.3 任务完成

1. 写入 COMPLETED
2. 正常完成后删除进度文件

## 四、代码修改

### 4.1 进度工具函数

在 `neuracle/rabbitmq/progress.py` 中已实现：

```python
from enum import IntEnum

class ModelProgress(IntEnum):
    START = 0
    PREPARE_T1_DONE = 10
    PREPARE_T2_DONE = 20
    DENOISE_DONE = 35
    INIT_ATLAS_DONE = 50
    SEGMENTATION_DONE = 70
    SURFACES_DONE = 85
    COMPLETED = 100

def load_progress(progress_file: Path) -> int:
    """加载进度，返回当前百分比，未找到则返回 0"""
    if progress_file.exists():
        return int(progress_file.read_text().strip())
    return 0

def save_progress(progress_file: Path, progress: int) -> None:
    """保存进度"""
    progress_file.write_text(str(progress))
```

### 4.2 Model 任务代码修改

#### 进度判断逻辑

只有当消息的 `redelivered` 属性为 `True` 时（表示 RabbitMQ 自动重发的消息），才从进度文件恢复。否则从头开始。

```python
# 在 handle_message 中判断
method_redelivered = method.redelivered

# 传递给任务处理函数
task_handler(message_queue, task_id, params, redelivered)
```

#### Model 任务

进度文件位置：`subject_dir / ".progress.txt"`

```python
def handle_model_task(..., redelivered: bool = False):
    subject_dir = get_subject_dir(params.dir_path)
    ensure_data_root()
    subject_dir.mkdir(parents=True, exist_ok=True)

    progress_file = subject_dir / ".progress.txt"
    if redelivered:
        current_progress = load_progress(progress_file)
        logger.info("任务 %s 为重发消息，从进度 %d 恢复", task_id, current_progress)
    else:
        current_progress = 0

    # 下载文件...

    # PREPARE_T1_DONE
    if current_progress < ModelProgress.PREPARE_T1_DONE:
        prepare_t1(...)
        save_progress(progress_file, ModelProgress.PREPARE_T1_DONE)
    send_progress(..., ModelProgress.PREPARE_T1_DONE)

    # ... 其余步骤同上
```

## 五、进度文件清理

- 任务正常完成（COMPLETED）后删除进度文件
- 任务异常时不删除，用于下次续跑
