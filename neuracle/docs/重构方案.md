# Neuracle 代码重构方案

## 一、现状分析

### 1.1 当前问题

现有代码存在以下主要问题：

1. **main.py 过于臃肿**：768行，混杂了消息处理、任务调度、进度汇报等多种职责
2. **模块边界不清晰**：`utils` 模块承载了太多不相关的功能
3. **rabbitmq 模块职责混杂**：消息基础设施与业务参数处理混在一起
4. **配置管理分散**：`.env` 配置分散在多个模块中读取
5. **缺乏统一的错误处理**：各模块错误处理方式不一致
6. **进度汇报逻辑分散**：进度枚举、消息构建、发送逻辑分散在多个文件

### 1.2 当前架构

```
neuracle_backup/
├── main.py                  # 768行, 中心调度器
├── rabbitmq/               # 消息基础设施
│   ├── listener.py
│   ├── sender_thread.py
│   ├── schemas.py
│   ├── validator.py
│   ├── progress.py
│   ├── task_handlers.py
│   └── message_builder.py
├── charm/                   # 头部模型生成
├── ti_simulation/           # TI正向仿真
├── ti_optimize/             # TI逆向优化
├── logger/                  # 日志配置
├── oss_tool/                # OSS上传下载
├── mesh_tools/              # 网格格式转换 (已移除，无需保留)
├── utils/                   # 共享工具(过于臃肿)
└── atlas/                   # 脑图谱数据
```

---

## 二、重构目标

1. **单一职责原则**：每个模块有清晰定义的职责
2. **可维护性**：代码组织清晰，易于理解和修改
3. **可扩展性**：新功能可以方便地添加
4. **配置集中化**：配置管理统一、规范

---

├── ti_optimization/            # 优化模块
├── charm/                   # 头部模型模块

├── config/                  # 配置模块

├── logger/                  # 日志模块
├── utils/                   # 工具模块
├── demo/                    # 示例代码
